Running the Demo

Extract the contents of the zip file and then run;

npm install
npm start

Navigate to http://localhost:8080 to view the demo in your local environment.

Layout

Omega consists of 2 main parts; the application layout component and layout resources. application.html inside app folder is the html template for the base layout having a router-outlet for routing. Required resources for layout are placed inside the resources/layout folder. To install omega, use the application.html content as your main template and copy the layout resources to your application to get the styling and functionality. CSS and JS resources that belong to layout are loaded with link-script tags in index.html of sample application however you may also use a bundler like webpack to install them as well.

If you have an existing CLI application, see the CLI section below for integration.

Initialize

After the view of your application is initialized, call init method by passing the native element to initiate the layout.

import {Component,AfterViewInit,ElementRef} from '@angular/core';

declare var Omega: any;
    
@Component({
    selector: 'my-app',
    templateUrl: 'app/application.html'
})
export class Application implements AfterViewInit {

    constructor(private el: ElementRef) {}

    ngAfterViewInit() {
        Omega.init(this.el.nativeElement);
    }
}

Dependencies

  • Angular 2.0+
  • PrimeNG 1.0+
  • jQuery for Layout
  • Nanoscroller for menu
  • Animate.css for animations

Icons

Omega Layout uses font awesome icons for layout so enable font awesome support in PrimeFaces by setting primefaces.FONT_AWESOME context parameter in web.xml as true. If your PrimeFaces version does not have the font awesome integration, include the font-awesome manually in your application.

Default icons of Omega theme are png based and Omega Layout enhances them using sharp looking font awesome icons as well.

Cards

Card is a section to group content and layout provides a built-in css for it. Apply .card style class to your container to use it.

<div class="card">
    Content here
</div>

<div class="card card-w-title">
    <h1>Title</h1>
    Content here
</div>

Grid CSS

Omega uses new PrimeFaces Grid CSS throughout the samples, we strongly suggest using Grid CSS as your layout framework as it is well tested and supported by PrimeFaces. Grid CSS is automatically included on newer versions however if your PrimeFaces version is older than 5.3.14, add the provided grid.css file under /omega-layout/css/ folder to your template manually.

SASS Customization

Omega is implemented using sass where sections are organized in different scss files under sass folder such as _dashboard.scss, _login.scss, _main.scss. The _variables.scss file contains common parameters like fontSize, colors and more. In order to customize the layout, you need to make changes to the .scss files and generate a css file called layout.css.

Angular CLI Integration

If you have an existing application, skip to the "Integration with an Existing CLI Application" section. For creating a project with using angular-cli, we need to download and install the angular-cli.

npm install -g angular-cli

Creating an Empty Project with Angular CLI

Run the following commands to create the application and run it.

ng new omega-with-cli
cd omega-with-cli
ng serve

Now install PrimeNG.

npm install primeng --save

Locate the styles.css inside the src folder and add the css of PrimeNG.

@import url('../node_modules/primeng/resources/primeng.min.css');

For a sample quickstart on PrimeNG with CLI, visit primeng-quickstart-cli at GitHub.

Integration Omega with an Existing CLI Application

We need to copy the resources to the CLI project. First step is the main layout for Omega located in the app folder. Copy the content of omega/app/application.html and omega/app/application.ts to cli applications src/app/app.component.html and app.component.ts files respectively.

Next step is to install the assets of Omega. Copy the resoures/layout folder to app/assets folder of your application, then locate angular-cli.json at the root of your cli application and add following styles and scripts. 

"styles": [
        //...
        "./app/assets/layout/animate.css",
        "./app/assets/layout/nanoscroller.css",
        "./app/assets/layout/layout.css"
      ]

"scripts": [
        "../node_modules/jquery/dist/jquery.min.js",   
        "./app/assets/layout/js/layout.js",
        "./app/assets/layout/js/nanoscroller.js"
      ]

Layout

In CLI, the main application component bootstraps the layout.

import { Component, AfterViewInit, ElementRef } from '@angular/core';

declare var Omega: any;

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements AfterViewInit {

  constructor(private el: ElementRef) {}

  ngAfterViewInit() {
        Omega.init(this.el.nativeElement);
    }
}

Run

Run ng-serve to test the setup.
